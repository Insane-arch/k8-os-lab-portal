About the project

Why this cluster exists

This portal was built as a serious Operating Systems Lab project to connect theory with real container orchestration. It shows how a Kubernetes cluster acts like a distributed operating environment managing processes, storage, networking, and recovery across multiple machines.

Custom Full Stack App

A custom-built Node.js application with a modern frontend, real database connectivity, and persistence testing instead of just a basic demo page.

Stateful Architecture

MySQL runs with persistent storage, which means saved messages remain available even after pod deletion or recreation.

Operating System Concepts

Process management, scheduling, networking, access control, monitoring, resource handling, and fault recovery are all visible in one project.

Core capabilities

What this deployment demonstrates

⚙

Pod Scheduling

Kubernetes schedules workload pods on worker nodes based on the deployment rules and labels.

🛡

RBAC Security

Role-Based Access Control protects cluster visibility and dashboard usage with controlled permissions.

🗄

Persistent Storage

MySQL uses PersistentVolume and PersistentVolumeClaim so data survives restarts and recreations.

🌐

Service Networking

Internal service communication allows the application to reach the database through Kubernetes DNS.

📈

Scalability

The application runs with multiple replicas, making it easier to demonstrate availability and scaling.

♻

Self-Healing

If a pod is deleted, Kubernetes recreates it automatically to maintain the desired state.
